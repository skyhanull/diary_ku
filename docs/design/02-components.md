# 컴포넌트 규칙

## 버튼

### Primary
- `Cedar` 계열을 사용한다
- 필요하면 아주 약한 그라데이션 허용
- 텍스트는 충분한 대비를 유지한다
- 클릭 시 아주 미세한 눌림감을 준다

### Secondary
- `Oatmeal` 또는 밝은 표면 색상 사용
- 과한 border 없이 톤 차이로 구분
- 본문보다 약간 강한 무게감만 준다

### Tertiary
- 텍스트 중심 액션
- hover에서 은은한 배경 변화만 준다
- 링크와 구분되도록 클릭 가능한 영역은 확보한다

## 입력창
- 기본 입력창은 과하게 박스형으로 만들지 않는다
- 낮은 대비 border 또는 배경 변화 중심으로 구성한다
- 포커스 시 배경이 약간 열리는 느낌을 준다
- 로그인 등 명확성이 중요한 폼은 너무 실험적으로 만들지 않는다

## 카드
- 카드 분리는 그림자보다 배경색 차이와 얇은 border로 해결한다
- 카드 내부는 숨 쉬는 듯한 여백을 준다
- 정보가 많아도 촘촘하게 압축하지 않는다

## 리스트
- divider line 남용을 피한다
- 아이템 간 간격과 배경 변화로 구분한다
- 선택 상태는 따뜻한 강조 배경으로 표현한다

## 모달 / 팝오버 / 플로팅 패널
- 떠 있는 요소는 가벼운 ambient shadow를 사용한다
- 배경은 `Vellum` 계열로 유지한다
- 필요하면 아주 약한 blur를 사용하되 과하지 않게 한다

## 아이콘
- 기본적으로 `Stone Gray` 또는 `Raw Umber` 사용
- 순수 검정은 피한다
- 아이콘은 장식보다 정보 전달 역할을 우선한다

## 빈 상태
- 충분한 여백을 사용한다
- 짧은 설명 + 명확한 CTA 하나를 둔다
- 허전함이 아니라 의도된 정적처럼 보여야 한다
