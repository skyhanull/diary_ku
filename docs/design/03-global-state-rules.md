# 글로벌 상태 규칙

## 원칙
상태 변화는 조용하고 촉감 있게 느껴져야 한다.
피드백은 분명해야 하지만, 거칠거나 기술적으로 보여서는 안 된다.

## 기본 상태
- 기본 배경은 `Vellum` 또는 `Oatmeal` 사용
- 주요 텍스트는 `Raw Umber`
- 보조 텍스트와 아이콘은 `Stone Gray`
- 경계선은 얇고 낮은 대비 유지

## 호버 상태
- 강한 색 변화보다 부드러운 명도 변화 사용
- 버튼과 카드는 살짝 더 깊은 톤으로 반응
- 과한 glow, 네온 효과, 강한 outline 금지

## 활성 / 눌림 상태
- 실제로 눌리는 듯한 촉감을 준다
- `scale-98` 또는 `scale-95` 정도의 미세한 축소 사용
- 전환은 `150ms ~ 200ms ease-out`
- 강한 그림자보다 배경 톤을 조금 더 눌러서 표현

## 포커스 상태
- 접근성은 지키되 부드러운 인상을 유지한다
- `Cedar` 계열 outline을 낮거나 중간 투명도로 사용
- 브라우저 기본 파란 포커스 링 같은 인상은 가능한 한 피한다

## 선택 상태
- 과한 대비보다 톤 차이로 강조한다
- 카드, 캘린더 날짜, 사이드 패널은 선택 여부가 명확히 보여야 한다
- 공격적인 강조보다 따뜻한 활성 상태를 목표로 한다

## 비활성 상태
- 완전히 흐려지지 않게 유지한다
- 클릭 불가라는 점은 분명해야 한다
- 대비와 시각적 무게를 조금만 낮춘다

## 로딩 상태
- 차분한 페이드, 스켈레톤, 은은한 shimmer를 우선 사용한다
- 필요 이상으로 요란한 스피너는 피한다
- 사용자가 기다리는 이유를 이해할 수 있어야 한다

## 성공 상태
- `Sage` 사용
- 안심되는 피드백을 목표로 한다
- 과한 축하형 모션은 지양한다

## 에러 / 삭제 상태
- `Dusty Rose` 사용
- 강한 경고보다는 차분하지만 분명한 표현
- 꽉 찬 빨강 배경보다 옅은 틴트와 읽기 쉬운 텍스트를 우선

## 전환 규칙
- 대부분의 상태 변화는 `200ms ~ 300ms ease-in-out`
- 갑작스럽거나 기계적인 움직임 금지
- 종이를 다루는 듯한 차분한 감각을 목표로 한다
