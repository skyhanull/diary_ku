# Do / Don't

## Do
- 따뜻한 종이 계열 톤을 유지한다
- 충분한 여백으로 화면이 숨 쉴 수 있게 만든다
- 아이콘과 보조 텍스트는 부드러운 색으로 눌러준다
- 작은 촉감형 인터랙션을 사용한다
- 톤 차이로 레이어를 만든다
- 감성과 사용성 사이의 균형을 유지한다

## Don't
- 순수 검정 `#000`을 텍스트 기본색으로 사용하지 않는다
- 대시보드형 SaaS 느낌으로 과하게 구조화하지 않는다
- 과한 그림자, 네온 포커스, 차가운 유리 질감을 남용하지 않는다
- 정보 밀도가 높은 관리자 도구처럼 보이게 만들지 않는다
- 감성을 위해 캘린더와 에디터의 사용성을 희생하지 않는다

## 이 정도면 충분한가
현재 문서 세트는 MVP 단계 디자인 시스템으로는 충분히 탄탄한 편이다.
특히 아래가 이미 갖춰져 있다.

- 브랜드 무드
- 색상 역할
- 타이포 방향
- 표면 / 경계 / 그림자 원칙
- 버튼 / 입력 / 카드 규칙
- 글로벌 상태 규칙
- 캘린더 / 에디터 예외 규칙

## 아직 남은 확장 가능 항목
프로젝트가 더 커지면 아래를 추가하면 된다.

- 실제 토큰 이름과 CSS 변수 매핑
- 캘린더 셀 상태별 상세 규칙
- 에디터 툴바 / 패널 / 속성시트 상세 규칙
- 모바일 반응형 규칙
- 아이콘 스타일 가이드
- 빈 상태 / 에러 상태 카피 톤 가이드
