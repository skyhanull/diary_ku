# 캘린더와 에디터 예외 규칙

## 왜 예외가 필요한가
DearMe는 감성적인 서비스지만, 메인 화면의 달력과 에디터는 구조와 조작이 핵심이다.
따라서 브랜드 무드를 유지하되, 이 두 화면에서는 사용성과 명확성을 우선한다.

## 캘린더 규칙
- 메인 화면은 감성적 랜딩이 아니라 날짜 선택 허브다
- 날짜 셀의 구분감은 분명해야 한다
- 선택된 날짜, 작성된 날짜, 오늘 날짜를 명확히 구분한다
- 너무 장식적인 배경 때문에 날짜 숫자가 묻히면 안 된다
- 달력은 안정적인 격자 구조를 유지한다

## 캘린더 허용 예외
- 낮은 대비 구분선 허용
- 상태 배지나 작은 도트 표시 허용
- hover와 selected 상태를 일반 카드보다 더 명확하게 표현 가능

## 에디터 규칙
- 감성보다 조작 가능성이 먼저다
- 상단 액션 영역, 캔버스, 패널, 툴바는 역할이 분명히 구분되어야 한다
- 아이템 선택, 이동, 리사이즈 상태가 명확하게 보여야 한다
- 꾸미기 요소는 자유로워도 조작 UI는 혼란스럽지 않아야 한다

## 에디터 허용 예외
- 일반 화면보다 조금 더 뚜렷한 경계선 허용
- 선택 박스, 리사이즈 핸들, 활성 패널 강조를 더 명확하게 사용 가능
- 기능 버튼은 감성보다 식별성과 접근성을 우선

## 입력 관련 예외
- 다이어리 본문은 자연스럽고 열려 있는 입력 경험을 사용해도 된다
- 로그인, 검색, 설정 입력은 더 전통적인 폼 스타일을 허용한다
- 모든 입력창에 같은 미학을 강제하지 않는다
