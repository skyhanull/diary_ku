# 테마와 브랜드

## 브랜드 방향
Memolie의 기준 무드는 `디지털 헤리티지 저널`이다.
생산성 툴보다는 오래 간직하는 다이어리와 스크랩북의 중간 지점을 지향한다.

## 핵심 인상
- 따뜻함
- 종이 같은 질감
- 조용한 프리미엄
- 사적인 기록 공간
- 아기자기하지만 과하지 않은 디테일

## 피해야 하는 인상
- 기업형 SaaS 대시보드
- 차갑고 유리 같은 미래형 UI
- 보라색 중심 테마 편향
- 지나치게 키즈앱 같은 장난감 톤
- 정보 밀도가 높은 관리자 도구 같은 분위기

## 현재 기본 테마
현재 코드베이스는 사실상 단일 기본 테마를 사용한다.
구현은 HSL 기반 semantic CSS 변수와 Tailwind 확장 색상 조합으로 구성된다.

### 전역 배경
- `body`는 `bg-background`
- 상단 우측에서 퍼지는 cedar 계열 radial gradient를 얹는다
- 전체 인상은 밝고 따뜻한 종이 계열이다

### 기본 표면
- 메인 카드: `bg-card`
- 보조 카드: `bg-oatmeal`, `bg-paper-warm`, `bg-paper-soft`
- overlay: 반투명 white 계열 + `backdrop-blur`

### 기본 포커스
- semantic `ring`
- 일부 에디터 선택 UI는 `--color-focus`를 사용한다

## 상태 테마
- 성공: `sage`
- 에러: `rose-pale` + `rose-danger`
- 보조 메시지: `oatmeal` + `cedar`

## 화면별 예외 규칙

### 홈 캘린더
- 감성보다 날짜 선택 허브 역할이 우선이다.
- 선택 날짜, 오늘, 기록 있는 날짜, 일정 있는 날짜를 분명히 구분해야 한다.
- 낮은 대비 선과 카드 레이어링은 허용되지만 격자 안정성을 깨면 안 된다.

### 에디터
- 조작 명확성이 최우선이다.
- 툴바, 캔버스, 패널, 저장 상태가 한눈에 구분되어야 한다.
- 선택 박스, 리사이즈 핸들, 활성 패널 강조는 일반 화면보다 더 선명해도 된다.

### 보관함
- 종이 카드 감성은 유지하되 검색성과 스캔 속도가 중요하다.
- 카드 variation은 허용하지만 정보 구조는 일관돼야 한다.

### 공유 편지
- 현재 share 화면은 가장 장식적인 영역이다.
- 그라데이션, 편지지 질감, theme별 색상 예외가 존재한다.
- 다만 읽기성과 본문 집중은 계속 우선해야 한다.

## 새 토큰 또는 테마 추가 절차
1. 장식값인지 반복되는 시스템 값인지 구분한다.
2. 반복되는 값이면 `tailwind.config.ts` 또는 `globals.css` semantic 변수에 추가한다.
3. 기존 컴포넌트 두세 곳 이상에서 재사용될지 확인한다.
4. 문서의 `01-tokens.md`와 `03-components.md`를 함께 갱신한다.
5. 새 화면에서만 필요한 예외면 토큰 추가보다 패턴 문서화로 끝낸다.

## 새 화면 작업 전 체크
- 이 화면은 홈/에디터/보관함/공유 중 어느 화면군에 속하는가
- 기본 warm theme를 유지하면서도 역할 명확성이 확보되는가
- 감성을 이유로 조작 가능성, 가독성, 대비를 해치지 않았는가
