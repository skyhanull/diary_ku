# 보관함 검색 알고리즘

보관함 검색은 저장된 일기를 단순히 필터링하는 대신, 검색어와 기록의 관련도를 점수로 계산해 정렬합니다. 사용자가 찾고 싶은 기록을 더 빨리 찾도록 제목, 태그, 감정, 본문, 날짜를 서로 다른 중요도로 평가합니다.

## 해결하려는 문제

일기 보관함에서는 기록이 많아질수록 날짜만으로 다시 찾기 어렵습니다. 같은 단어가 본문에 한 번 들어간 기록보다, 제목이나 태그에 직접 들어간 기록이 더 관련도 높게 보여야 합니다.

예를 들어 `여행 행복`을 검색했을 때는 아래 기록을 우선 노출하는 것이 자연스럽습니다.

- 제목에 `여행`이 들어간 기록
- 태그에 `여행`이 붙은 기록
- 감정이 `좋음`, `행복`, `기쁨` 계열인 기록
- 본문에 두 검색어가 모두 들어간 기록
- 관련도가 같다면 더 최근에 작성한 기록

## 전체 흐름

1. 검색어를 정규화합니다.
2. 검색어를 단어 단위로 나눕니다.
3. 각 일기를 검색용 인덱스로 변환합니다.
4. 제목, 태그, 감정, 본문, 날짜 필드별 점수를 계산합니다.
5. 여러 검색어가 모두 매칭되면 보너스 점수를 추가합니다.
6. 관련도 점수순으로 정렬하고, 점수가 같으면 최신순으로 정렬합니다.
7. 검색 결과 카드에 `제목 일치`, `태그 일치`, `본문 일치` 같은 매칭 이유를 표시합니다.

## 검색어 정규화

검색어와 저장된 텍스트는 비교 전에 `normalizeSearchText`로 정리합니다.

```ts
export function normalizeSearchText(value: string) {
  return value
    .normalize('NFKC')
    .toLowerCase()
    .replace(/[^\p{L}\p{N}\s-]/gu, ' ')
    .replace(/\s+/g, ' ')
    .trim();
}
```

이 처리를 통해 대소문자, 중복 공백, 일부 특수문자 차이를 줄입니다. 예를 들어 `Happy!! 여행`은 `happy 여행`처럼 비교 가능한 형태가 됩니다.

## 검색 인덱스

각 일기는 바로 비교하지 않고 검색용 인덱스로 변환합니다.

```ts
interface SearchIndex {
  title: string;
  body: string;
  tags: string[];
  date: string;
  mood: string;
}
```

각 필드의 역할은 다릅니다.

- `title`: 사용자가 직접 붙인 제목이라 가장 중요한 신호입니다.
- `tags`: 사용자가 직접 분류한 키워드라 본문보다 높은 신호입니다.
- `mood`: 감정 점수를 검색 가능한 한국어 키워드로 확장합니다.
- `body`: 실제 일기 본문입니다.
- `date`: `2026-04-12`, `20260412` 형태 검색을 보조합니다.

## 감정 키워드 확장

감정은 저장된 점수만으로는 검색하기 어렵기 때문에 검색용 텍스트로 확장합니다.

| 감정 점수 | 검색 키워드 |
| --- | --- |
| `100` | `좋음 행복 기쁨 밝음` |
| `80` | `평온 차분 안정` |
| `60` | `보통 무난` |
| `40` | `흐림 우울 다운` |
| `20` | `슬픔 눈물 속상` |

이렇게 하면 사용자가 `행복`이라고 검색해도 `좋음` 감정의 기록이 검색 결과에 포함될 수 있습니다.

## 필드별 점수

현재 검색 가중치는 아래 기준으로 계산합니다.

| 필드 | 완전 일치 | 시작 일치 | 포함 |
| --- | ---: | ---: | ---: |
| 제목 | `120` | `86` | `58` |
| 태그 | `100` | `72` | `42` |
| 감정 | `46` | `34` | `26` |
| 본문 | `34` | `26` | `16` |
| 날짜 | - | - | `18` |

제목과 태그 점수를 높게 둔 이유는 사용자가 의도적으로 작성한 요약 정보이기 때문입니다. 본문은 길이가 길어 우연히 단어가 포함될 수 있으므로 상대적으로 낮은 점수를 줍니다.

## 보너스 점수

검색어가 여러 단어일 때 모든 단어가 하나라도 매칭되면 `32`점을 추가합니다.

예를 들어 `여행 행복` 검색 시:

- `여행`만 포함한 기록보다
- `여행`과 `행복`이 모두 매칭된 기록이 더 위로 올라갑니다.

## 최신성 보정

검색 결과의 기본 정렬은 관련도 점수지만, 같은 점수라면 최신 기록을 먼저 보여줍니다. 검색어가 없을 때는 최신순으로 정렬합니다.

검색어가 있을 때도 최신 기록에 작은 보정 점수를 부여합니다. 다만 제목이나 태그 매칭보다 훨씬 작은 점수라서, 최신성만으로 관련도 높은 기록을 밀어내지 않도록 했습니다.

## 결과 표시

검색 결과에는 최대 3개의 매칭 이유를 함께 반환합니다.

```ts
export interface ArchiveSearchResult {
  entry: DiaryEntrySummary;
  score: number;
  reasons: string[];
}
```

예시:

```ts
{
  entry,
  score: 208,
  reasons: ['제목 일치', '태그 일치', '검색어 모두 포함']
}
```

화면에서는 이 이유를 카드 칩으로 보여줍니다. 사용자가 왜 이 기록이 위에 나왔는지 바로 이해할 수 있게 하기 위한 장치입니다.

## 이력서/면접 설명 포인트

아래처럼 설명할 수 있습니다.

> 보관함 검색에서 단순 문자열 필터링만 사용하면 본문에 우연히 단어가 들어간 기록이 제목이나 태그가 일치하는 기록보다 위에 노출되는 문제가 있었습니다. 이를 개선하기 위해 제목, 태그, 감정, 본문, 날짜를 검색 인덱스로 분리하고 필드별 가중치를 다르게 적용한 관련도 점수 기반 랭킹 로직을 구현했습니다. 또한 여러 검색어가 모두 매칭될 때 보너스 점수를 주고, 동점일 때는 최신순으로 정렬해 사용자가 원하는 기록을 더 자연스럽게 찾을 수 있도록 개선했습니다.
