# DearMe PRD (MVP)

## 1. 제품 한 줄 정의
스티커/텍스트/사진으로 하루를 꾸미고, 달력 기반으로 쌓아가는 디지털 다이어리 웹앱.

## 2. 타겟 사용자
- 다이어리 꾸미기를 좋아하는 10~30대
- 기록보다 "꾸미는 재미"가 있어야 꾸준히 사용하는 사용자
- 종이 다이어리의 휴대/보관 불편을 느끼는 사용자

## 3. 문제 정의
- 기존 메모앱은 감성 꾸미기 자유도가 낮음
- SNS는 캘린더 기반 기록/회고 구조가 약함
- 종이 다이어리는 수정, 공유, 검색이 불편함

## 4. MVP 목표
- 핵심 경험 검증: "하루 페이지를 꾸미고 저장/공유할 수 있다"
- 기간: 10주

## 5. MVP 핵심 기능
1. 소셜 로그인 1개(구글)
2. 월간 캘린더 + 일별 페이지 생성/수정/삭제
3. 꾸미기 에디터
   - 배경/패턴
   - 스티커 배치(이동/크기/회전/레이어)
   - 텍스트 박스(폰트/색상)
   - 사진 업로드(최대 5장)
4. 공개/비공개 + 읽기 전용 공유 링크

## 6. 비범위(초기 제외)
- 실시간 협업
- 스티커 마켓 결제
- 고급 손그림 브러시
- 완전 오프라인 모드

## 7. 성공 지표
- 첫 작성 후 7일 내 재방문율 25%+
- 활성 사용자 기준 페이지 저장 비율 40%+
- 작성 페이지 중 공유 사용률 15%+

## 8. 화면 구조
1. 온보딩/로그인
2. 홈(월간 캘린더)
3. 데일리 보기
4. 에디터
5. 마이페이지

## 9. 기술 스택
- Frontend: Next.js + TypeScript + Tailwind
- BaaS: Supabase(Auth, Postgres, Storage, RLS)
- Canvas: Konva.js 또는 Fabric.js

## 10. 리스크와 대응
- 에디터 성능: 이미지 압축, 아이템 제한, 저장 구조 최적화
- 저작권: 기본 스티커 에셋 라이선스 명확화
- 리텐션: 온보딩 템플릿 + 작성 프롬프트 제공
