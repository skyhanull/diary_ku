# 스크린샷 캡처 체크리스트

README 상단에서 참조하는 이미지들을 이 폴더에 넣어주세요. **파일명을 그대로 맞추면** README에 자동으로 표시됩니다.

| 파일명 | 화면 | 캡처 포인트 |
| --- | --- | --- |
| `home.png` | 홈 대시보드 (`/`) | 감정이 표시된 캘린더 + 우측 날짜/일정·감정 카드 + 하단 운세/날씨가 한 화면에 보이게 |
| `editor.png` | 에디터 (`/editor/[date]`) | 종이 위에 본문 + 텍스트/스티커/이미지 아이템이 배치된 상태. 왼쪽 툴레일과 오른쪽 패널도 함께 |
| `chat.png` | AI 일기 친구 채팅 | 과거 일기를 근거로 답변하는 대화 장면 (RAG가 드러나는 답변이면 베스트) |

## 팁
- **해상도**: 가로 1280~1440px 권장. 브라우저 창을 적당히 줄여 여백이 과하지 않게.
- **GIF도 가능**: `home.gif`처럼 확장자만 바꿔 넣고 README의 파일명도 함께 바꾸면 됩니다. 캔버스 드래그/회전, AI 스트리밍 응답은 GIF가 훨씬 효과적입니다.
  - macOS: [Kap](https://getkap.co) 또는 [Gifox]로 화면 녹화 → GIF 저장.
- **데모 데이터**: "테스트 계정으로 시작"으로 로그인하면 채워진 상태로 캡처하기 좋습니다. 빈 화면보다 실제 기록이 있는 화면이 훨씬 설득력 있습니다.
- **용량**: PNG는 8~10장 넘지 않게, GIF는 각 5MB 이하로. 무거우면 [tinypng.com](https://tinypng.com)이나 `gifsicle -O3`로 압축.

## 넣은 뒤
1. `docs/screens/`에 3개 파일을 추가하고 커밋합니다.
2. README 상단의 **라이브 데모 URL**(`https://your-app.vercel.app` 자리)을 실제 Vercel 배포 주소로 바꿉니다.
3. GitHub에서 README가 이미지와 함께 렌더되는지 확인합니다.
